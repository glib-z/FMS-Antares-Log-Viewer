    Порядок работы:
    1. Открыть файл протокола СКЗ - во вкладке "FMS log" отображается протокол СКЗ в исходном виде (дата и время контроллера стойки НА СКЗ).
    2. Время для команды PROGRAM START подтягивается из протокола, дата из заголовка протокола. Если ПСП переходит через полночь, дата команды PROGRAM START может отличаться от даты протокола. Для приведения времени протокола ко времени UTC необходимо ввести дату и время UTC выдачи команды PROGRAM START в формате "dd.mm.yyyy / hh:mm:ss.zzz". Время IGNITION определяется из протокола относительно времени команды на снятие питания с бортовой аппаратуры. Относительно времени ЗДУ отсчитывается время Т- (countdown). Для уточнения времени Т- ввести дату и время ЗДУ в формате "dd.mm.yyyy / hh:mm:ss.zzz".

    Результаты обработки:
    - во вкладке "Report" отображается протокол без параметра "Network control" и значений регистров, исключены строки с повторяющимися значениями параметров (по параметру "Average temperature" порог 0.02), добавлено время UTC и время T- (countdown), выведены времена команд для подготовки отчёта;
    - во вкладках "Level F1" и "Level O1" выводятся графики изменения значений уровней заправки в баках ОКПС для времени T- (countdown), длительность интервала ограничена временем начала изменения уровнеей и временем ЗДУ;
    - во вкладках "Level F1 (UTC)", "Level O1 (UTC)" и "Temperature (UTC)" выводятся графики для времени UTC, длительность интервала соответсвует длительности работы бортовой аппаратуры.
